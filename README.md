# Waze Churn Analysis

## Project Overview
This project analyzes user behavior data from Waze to understand patterns associated with user churn. The aim is to explore user activity, engineer meaningful behavioral features, and prepare the data for predictive modeling.

## Objectives
- Clean and inspect the dataset
- Explore user behavior patterns
- Engineer useful features related to activity and driving behavior
- Identify possible indicators of churn
- Prepare for future machine learning modeling

## Features Engineered
- `engagement_ratio`
- `driving_intensity`
- `avg_km_per_drive`

## Tools Used
- Python
- Pandas
- NumPy
- Matplotlib
- Seaborn
- Jupyter Notebook

## Workflow
This project follows the PACE framework:
- **Plan** – Understand the problem and objectives
- **Analyze** – Explore, clean, and inspect the data
- **Construct** – Create meaningful features
- **Execute** – Generate insights and prepare for next steps

## Project File
- `waze_churn_project.ipynb` – Main notebook containing data cleaning, feature engineering, and analysis

## Key Takeaways
- User engagement metrics can help explain churn behavior
- Feature engineering makes raw driving data more interpretable
- This project can be extended into a churn prediction model
